
# CONTEXT.MD 自动生成模板 (AI Instruction Template)
---
### **致AI助手 (Instructions for the AI Assistant):**
---

**你的角色:** 你是一名顶级的软件工程师和技术文档专家。

**你的任务:** 你将收到一份Python模块的完整源代码。请仔细阅读并深刻理解其设计和功能，然后严格按照下面的“最终输出格式”部分，生成一份高质量、清晰、准确的 `CONTEXT.MD` 文件。

**工作流程与分析指南:**

1.  **分析模块目标 (Module Goal):**
    * 阅读模块/文件的名称、核心类名（例如 `UTrackNetV1Backbone`）和关键函数名。
    * 用一句话高度概括这个模块的核心职责与目标。

2.  **分析核心设计思想 (Core Design Philosophy):**
    * **这是最重要的部分。** 请在代码中寻找关键的设计模式或架构思想。
    * **检查注册表/工厂模式:** 是否有 `Registry` 类、`build_*` 函数以及 `@...register_module` 装饰器？如果有，说明它是“配置驱动”和“可插拔”的。
    * **检查组件化思想:** 模型代码是否被拆分成了 `Backbone`, `Neck`, `Head` 等逻辑部分？
    * **检查核心算法:** 是否有实现某个特定的、关键的算法或逻辑？
    * 将这些发现总结为2-3个要点。

3.  **分析关键文件/代码导览 (Key File/Code Guide):**
    * 识别出该模块中最重要的类和函数。
    * 为每一个关键部分写一句注释，解释其作用。不要罗列所有函数，只选择最核心的。

4.  **分析如何使用 (How to Use):**
    * **优先寻找 `if __name__ == "__main__":` 代码块。** 这部分通常是作者留下的最佳使用示例和测试代码。
    * 将这个测试代码块提炼成一个最简短、最核心的用法示例。删除所有不必要的打印和断言，只保留核心的实例化和调用过程。

5.  **分析外部交互 (External Interactions):**
    * **分析 `import` 语句** 来确定其依赖关系。
    * **上游依赖 (Depends on):** 识别出它从哪些外部库（如`torch`）或其他项目模块（如`..builder`）导入了代码。
    * **下游消费 (Used by):** 根据模块的功能推断它会被谁使用。例如，一个 `Backbone` 模块会被一个 `Model` 的总装模块所使用。

### **新手使用指南**

1.  复制本文件 (`context_template.md`) 的全部内容。
2.  打开您需要为其生成文档的 Python 代码文件 (例如 `utracknetv1_backbone.py`)，复制其全部代码。
3.  将第1步和第2步的内容一起粘贴到大模型（如 Gemini, GPT-4 等）的对话框中。
4.  发送指令后，AI 将会自动生成格式化的 `CONTEXT.MD` 内容，复制并保存即可。
